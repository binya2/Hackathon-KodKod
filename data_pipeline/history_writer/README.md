# History & Archive Service (History Writer)

The **History & Archive Service** acts as the system's "Black Box" flight recorder. It persists the aggregated real-time World State into a NoSQL database for post-action debriefing, analysis, and auditing.

## Role in C4I Swarm
- **Long-term Persistence**: Saves the complete tactical picture (`WorldState`) generated by the `state_aggregator`.
- **Batch Processing**: Uses a buffered write mechanism to efficiently insert multiple records at once, minimizing database connection overhead.
- **Resilience**: Prevents data loss during network hiccups and guarantees that all states eventually reach the database.

## Kafka Topics
- **Consumes**:
  - `aggregated-state-topic`: The final fused output from the State Aggregator.
- **Produces**:
  - None.

## Database
- Connects to a **MongoDB** instance (typically at `mongodb://mongo:27017`).
- Appends an `archived_at` timestamp to every ingested document.

## Internal Logic Highlights
- Built using the high-performance `confluent-kafka` and `pymongo` libraries.
- **Smart Buffering**: Implements a dual-trigger flush mechanism. The buffer is flushed to MongoDB either when:
  1. It reaches **10 documents**.
  2. A **5-second timeout** passes (ensuring stale data isn't left in the buffer at the end of a session).
