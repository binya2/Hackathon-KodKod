# Log Aggregator Service

The **Log Aggregator** is a central logging microservice designed to collect and format system-wide operational logs generated by other microservices in the tactical drone swarm.

## Role in C4I Swarm
- **Centralized Telemetry Logging**: Acts as the single destination for human-readable debug and operational information.
- **Console Output Formatting**: Listens to raw JSON log payloads and formats them into clean, structured console outputs for monitoring and debugging.

## Kafka Topics
- **Consumes**:
  - `system.logs`: The dedicated topic where microservices (such as `attack_commander`) send their operational alerts, errors, and informational events.
- **Produces**:
  - None.

## Internal Logic Highlights
- Built using the high-performance `confluent-kafka` library.
- Extracts `timestamp`, `level` (INFO, WARN, ERROR), `service` name, and `message`.
- Runs a non-blocking fast polling loop to immediately print arriving logs to the terminal.
